# Quick Start
![Startup Login](assets/screenshots/startup-login.png)

Welcome to your new workspace. The Quick Start guide covers the essential steps to get your environment running, log in, and ensure everything is functioning correctly before you dive into deeper work.

## What you can do here
* Sign in or complete the first-run setup flow.
* Open the main chat workspace and confirm the app is responsive.
* Use the sidebar to move between chat, history, models, jobs, files, analytics, and settings.
* Check health, version, and runtime indicators before starting longer work.

## Typical workflow
To get started, navigate to the main web address and complete the initial login or setup process. Once authenticated, you will land in the main chat interface. Send a test message to verify that the default model is responding. Finally, take a moment to look at the sidebar and familiarize yourself with the available tools, and check the bottom corners for system health and version indicators.

## Key controls
* **Login Form:** Enter your credentials or complete the first-run setup.
* **Sidebar Navigation:** Switch between different sections of the application.
* **Chat Input:** Send your first test message to verify responsiveness.
* **Status Indicators:** Check system health, runtime, and version.

## Startup behavior

The first-run path explicitly guides you through setup. The runtime checks for an existing local environment before prompting you with download-source choices, and startup progress remains visible during initialization. Once the login or setup screen appears, verify a simple chat action and check your provider and model configurations before assuming the environment is fully ready.

## Notes and limits
* If the UI loads but chat does not respond, check runtime status, provider credentials, and logs before changing settings.
* Avoid pasting secrets into chat or screenshots; configure credentials through the intended settings or environment path.
