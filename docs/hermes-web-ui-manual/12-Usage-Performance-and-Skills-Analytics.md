# Usage and Analytics
![Usage Analytics](assets/screenshots/usage-analytics.png)

Usage and Analytics provide insights into how the system is performing, helping you track model activity, review costs, and understand skill utilization.

## What you can do here
* Review usage and cost trends.
* Inspect model and cache activity.
* Monitor performance over time.
* See which skills are used and when.

## Typical workflow
Periodically visit the analytics dashboard to review your usage patterns and associated costs. If you notice unexpected delays or spikes in activity, inspect the model and cache metrics to identify the source. Check the skills usage view to understand which automated procedures are triggered most frequently and optimize them if necessary.

## Key controls
* **Usage Dashboard:** View charts and trends related to overall system activity and cost.
* **Performance Metrics:** Monitor response times and cache hit rates.
* **Skills Analytics:** Track how often specific skills are invoked.
* **Date Filters:** Adjust the timeframe for the displayed data.
