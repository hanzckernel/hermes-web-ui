# Profiles
![Profiles Management](assets/screenshots/profiles-management.png)

Profiles allow you to maintain distinct, isolated contexts for your work, ensuring that credentials, configurations, and settings remain separated across different tasks or environments.

## What you can do here
* Create, clone, import, export, and switch profiles.
* Keep provider and channel settings isolated by profile.
* Review profile-specific status before starting work.

## Typical workflow
When switching to a different project or environment, navigate to the profiles section. Select an existing profile that matches your context or create a new one. If you need to duplicate a setup for experimentation, clone an existing profile to avoid altering production configurations. Ensure that the active profile is correctly loaded before starting your tasks.

## Key controls
* **Profile List:** View all available profiles and their active status.
* **Create/Clone Button:** Start a new profile or duplicate an existing setup.
* **Switch Profile:** Change the current operational context.
* **Import/Export:** Share or back up profile configurations.

## Notes and limits
* Switching profiles can change credentials, models, gateways, skills, and memory behavior.
* Avoid editing production-like profiles when experimenting; clone first if unsure.
