# Interface Overview
![Chat Main Overview](assets/screenshots/chat-main-overview.png)

The Interface Overview provides a structural map of the application, helping you understand where everything is located and how the various components interact.

## What you can do here
* Understand the main navigation areas and what each one is for.
* Recognize which pages are for daily use and which are for administration.
* Know where to look when a feature depends on runtime, credentials, or external services.

## Typical workflow
When you first start using the application, review the overall layout. Identify the main chat area for your daily tasks, and locate the administrative sections in the sidebar for configuration and monitoring. Keep in mind that certain features require external credentials or specific runtime states; knowing the interface will help you quickly navigate to the right place when a dependency is missing.

## Key controls
* **Main Chat Area:** The primary workspace for interaction.
* **Sidebar:** Access point for history, models, jobs, and settings.
* **Settings & Admin:** Configure credentials and monitor system health.

## Current interface changes

The interface features clear page sidebars for chat and history, a clean desktop title bar in packaged builds, and consistent page-level navigation. Treat the sidebar as the primary way to move between workflows, and use page-specific sidebars when working through long conversations or history lists.

## Notes and limits
* Some pages can run commands, write files, schedule work, or connect external channels; review the action before confirming.
* If you are documenting or debugging behavior, use screenshots plus logs rather than relying on memory.
